Agent Memory

Two-tier memory: session context
+ long-term anomaly recall

🧠

Short-Term — agent_conversation

• Last 20 turns per session_id
• Loaded on every run_agent() call
• Saved after every end_turn
• Trimmed between turns — never mid-loop
• Enables multi-question context
📚

Long-Term — agent_memory

• Written by detect_anomalies tool
• 7-day retention with auto-purge
• Dedup: skip if same (code, platform) within 30 min
• Injected as context when user asks
   "has this happened before?"
